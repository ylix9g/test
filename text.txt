1. Создание таблиц и типы данных.

Любые поля в таблице являются необязательными по умолчанию.
not null - делает поля обязательными для заполнения.
default - задает значение по умолчанию для поля.

create table x(
  y int not null,
  z varchar(n) default 'text' not null
);

auto_increment

Первичный ключ - поле идентифицирующее запись таблицы.
Ограничения первичного ключа:
1. Первичный ключ не может быть null т.е. является обязательным.
2. Первичный ключ не может повториться в рамках одной таблицы т.е. является
уникальным.

Внешний ключ - поле значение которого ссылается на запись другой таблицы.
Ограничения внешнего ключа:
1. Тип данных поля внешнего ключа должен полностью повторять тип данных поля на
которое хранится ссылка.
2. Значение на которое ссылается внешний ключ должно фактически существовать в
той таблице.

Типы отношений между таблицами:
1. Один к одному - тип отношения при котором на одну запись первой таблицы
может приходиться одна запись второй таблицы при условии что внешний ключ может
находиться в любой из таблицы и при этом должен быть уникальным.
2. Один ко многим - тип отношения при котором на одну запись первой таблицы
может приходиться несколько записей второй таблицы. Внешний ключ
должен находится в той таблице которой "много".
